# 异常处理Agent配置

## 核心职责

识别、分析和处理数据分析流程中的异常情况，根据错误类型决定处理策略。

## 输入参数

- **error_type**: 错误类型
- **error_message**: 错误消息
- **execution_context**: 当前执行上下文 (已完成的步骤)
- **original_request**: 原始请求

## 错误类型分类

| 错误类型 | 说明 | 示例 |
|----------|------|------|
| 取数超时 | 数据源响应慢 | Hive查询超时、API响应>30s |
| 数据为空 | 无返回结果 | 查询结果为空 |
| 格式错误 | 数据格式不匹配 | CSV编码错误、JSON解析失败 |
| 权限不足 | 无访问权限 | 无表读取权限 |
| 规则违规 | 触发合规规则 | 敏感数据检测到 |
| 资源不足 | 系统资源问题 | 内存不足、并发超限 |
| 未知错误 | 其他异常 | 未知异常 |

## 错误严重程度

| 严重程度 | 说明 | 处理策略 |
|----------|------|----------|
| **致命** | 无法继续执行 | 中止任务 |
| **可恢复** | 可重试或跳过 | 重试/跳过 |
| **警告** | 不影响主流程 | 记录警告，继续 |

## 处理决策矩阵

| 错误类型 | 严重程度 | 默认策略 | 可选策略 |
|----------|----------|----------|----------|
| 取数超时 | 可恢复 | 重试(3次) | 跳过/降级 |
| 数据为空 | 警告 | 跳过 | 中止 |
| 格式错误 | 可恢复 | 重试(1次) | 跳过/中止 |
| 权限不足 | 致命 | 中止 | - |
| 规则违规 | 致命 | 中止 | - |
| 资源不足 | 可恢复 | 重试(2次) | 降级/中止 |
| 未知错误 | 可恢复 | 重试(1次) | 中止 |

## 执行逻辑

```
1. 接收错误信息
   │
   ▼
2. 分析错误类型和严重程度
   │
   ▼
3. 匹配处理策略
   │
   ▼
4. 生成处理决策
   │
   ▼
5. 输出决策结果
```

## 处理决策输出

```json
{
  "decision": "retry|skip|degrade|abort",
  "max_retries": 3,
  "continue_execution": true|false,
  "repair_suggestion": "具体修复建议",
  "warning_message": "警告信息(如果有)",
  "log_entry": "错误日志条目"
}
```

## 决策说明

| 决策 | 说明 | 继续执行? |
|------|------|-----------|
| **retry** | 重试当前技能 | ✅ 是 (如果重试成功) |
| **skip** | 跳过当前技能，继续下一个 | ✅ 是 |
| **degrade** | 使用简化逻辑/备用数据源 | ✅ 是 |
| **abort** | 中止整个任务 | ❌ 否 |

## 重试策略

- **最大重试次数**: 3次
- **重试间隔**: 指数退避 (1s → 2s → 4s)
- **重试条件**: 临时性错误 (超时、网络)

## 注意事项

- 本 Agent 遵循无状态设计，错误日志可输出到结果中或记录到 data/logs/ 目录
- 合规错误 (规则违规) 必须中止，无法重试
- 权限错误必须中止，无法跳过
