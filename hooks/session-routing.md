# 智能路由规则规范

## 意图分类

| 意图代码 | 路由目标 | 典型特征 |
|----------|----------|----------|
| COMMAND | 主会话 | `/` 开头系统指令 |
| DATA_DEEP | Spawn danalyzer | 多步清洗/建模/绘图/看板 |
| RESEARCH | Spawn research | 研究报告/白皮书/调研/可行性评估/竞品分析/长文报告 |
| DATA_SIMPLE | 主会话 | 单次查询/简单聚合/取数 |
| GENERAL | 主会话 | 编程/Git/日常对话 |

冲突仲裁：深度分析优先。均为简单时按上表从上到下匹配。

## 7 步执行顺序

```
用户输入 → 1.意图分类 → 2.模糊检查 → 3.安全红牌检查 → 4.复杂度校验
  → 5.准备Spawn参数 → 6.Spawn子Agent → 7.验证输出
```

Step 1-5 之间禁止读取任何数据文件。

## 模糊检查（Step 2）

以下情况必须 AskUserQuestion 澄清，禁止猜测：
- NLU 置信度 < 0.7（查询过短/过模糊/缺关键信息）
- 命中安全红线但未达直接阻断级别
- 意图在 DATA_DEEP 与 RESEARCH 间模糊不清

**注意**: 「分析指引」表中的场景（预测无历史/指标未定义/大范围/伪分析）
意图明确（均为 DATA_DEEP），不属于模糊检查范围。
反问澄清在 Step 6 执行阶段进行，不在 Step 2 触发 AskUserQuestion。

## 上下文延续

- 输入含"继续/深化/再分析/然后"且上轮有明确路由目标 → 沿用上轮路由与参数
- 输入含显式新指令或话题变更 → 重置上下文，重新路由

## 安全红线（路由级拦截）

仅以下场景属于安全红线，命中后阻断当前流程：

| 红线 | 处置 |
|------|------|
| 写表/删库/修改生产配置 | 挂起，AskUserQuestion 二次授权 |
| 大批量数据导出/下载（全库/全表） | 挂起，AskUserQuestion 二次授权 |

## 分析指引（执行级注意事项）

以下场景**不改变路由**（按原始意图继续，均为 DATA_DEEP），具体处理方式见 `skills/danalyzer-core/SKILL.md` Section II「分析指引场景」表格。

> 路由层仅做意图分类，不维护执行细则。场景定义以 danalyzer-core 为单一事实源。

## 自动降级条件（满足任一即降级为 DATA_SIMPLE）

1. 仅单次 SQL 查询或简单聚合，无后续分析
2. 明确不需要图表/预测/建模
3. 上下文显示用户仅需快速结论

> **边界提示**：降级为 DATA_SIMPLE 的请求保留在主会话执行，不会 spawn danalyzer Agent，因此也不会触发知识检索（Step 2.5）。若查询含行业特征词但被误判降级，上下文注入会缺失。当前影响面极小（简单取数确实不需要知识检索），若有行业特定的简单取数需求，应提升为 DATA_DEEP。

## 主会话执行规则（DATA_SIMPLE / GENERAL 留在主会话时）

- 允许检索与读取数据文件进行简单查询或聚合
- 禁止多步清洗、建模、预测等深度分析
- 输出前必须检查并脱敏 PII 信息
- 安全红线同样适用于主会话执行

## 主会话执行边界（路由到子 Agent 后）

| 红线 | 说明 |
|------|------|
| 禁止数据探索 | 不得读数据文件、查表结构、预览数据 |
| 禁止预分析 | 不得执行任何数据计算或统计 |
| 禁止并行抢活 | 子 Agent 运行期间不得替它做任何工作 |
| 等待完成 | 子 Agent 返回前不发起新任务 |

## Spawn 验证（子 Agent 返回后）

| 检查项 | 通过条件 |
|--------|---------|
| 文件存在 | 输出文件路径存在且非空 |
| 不违规 | 输出中不包含 PII（粗略检查）|
| 非空结果 | 返回结果不为空或错误 |

验证失败：重新 Spawn 或告知用户。
