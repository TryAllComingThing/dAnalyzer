#  智能路由规则规范

## 1. 核心原则
* **语义优先**：基于自然语言理解（NLU）判断用户真实意图。
* **安全左移**：路由决策必须经过**程序化红牌检查**与**置信度评估**，高风险操作严禁自动执行。
* **最小特权与降级**：能主会话解决的绝不 Spawn 子 Agent；深度任务必须隔离执行；模糊意图必须降级或确认。

---

## 2. 意图分类与路由矩阵

根据复杂度与执行环境，将意图划分为三个层级。路由决策需严格遵循下表：

| 意图代码 | 路由目标 | 复杂度 | 典型特征与判定标准 |
| :--- | :--- | :--- | :--- |
| **COMMAND** | 主会话 | - | `/` 开头的系统指令。**优先级最高 (5)**。 |
| **DATA_DEEP** | **Spawn Agent** | 深度 | 需多步清洗/预测/绘图；产出 Dashboard/看板。**核心：复杂数据处理** |
| **RESEARCH** | **Spawn Agent** | 深度 | 需跨源检索、竞品分析、长文报告/PPT生成。**核心：产出文档/洞察** |
| **DATA_SIMPLE** | 主会话 | 简单 | 单次查询、简单聚合、取数。**核心：快、准** |
| **GENERAL** | 主会话 | - | 编程辅助、日常对话、知识问答等通用场景。 |

> **冲突仲裁原则**：当输入同时命中多个意图时，优先匹配**深度分析类**（DATA_DEEP/RESEARCH）；若均为简单类，按优先级数值（括号内）从高到低执行。

---

## 3. 执行流与降级策略

### 3.1 Spawn 隔离机制
仅 `DATA_DEEP` 和 `RESEARCH` 强制触发子 Agent。
* **输入**：用户原始请求 + 对应领域的指令集（如 `agents/danalyzer.md`）。
* **输出**：结构化结果。若子 Agent 发现任务超出能力范围，需返回 `[reroute: target_intent]` 信号。

### 3.2 自动降级逻辑（稳定性保障）
即便 NLU 判定为 `DATA_DEEP`，主会话在执行前必须进行**复杂度二次校验**。满足以下任一条件，**强制降级**为 `DATA_SIMPLE` 并在主会话执行：
1. 仅涉及单次 SQL 查询或简单聚合，无后续分析步骤。
2. 明确不需要图表、预测或复杂建模。
3. 上下文显示用户仅需一个快速结论。

---

## 4. 安全护栏 (Red Flags)

**此环节由主会话代码层强制执行，命中即拦截。**

| 红线类型 | 触发场景 | 处置动作 |
| :--- | :--- | :--- |
| **高风险操作** | 涉及写表、删库、修改生产环境配置 | **挂起**，强制要求用户二次授权（AskUserQuestion）。 |
| **伪分析** | 仅提供片段数据要求“深度分析” | **澄清**，引导用户补充原始数据或仅做简单解读。 |
| **范围失控** | 时间跨度极大（如“分析三年全量日志”） | **建议**，引导用户缩小范围或采用抽样分析。 |
| **依据缺失** | 要求预测但未提供历史数据或假设条件 | **驳回**，要求补充回溯窗口或前置假设。 |
| **口径模糊** | 关键指标未定义（如“用户数”指代不明） | **确认**，列出可选口径（DAU/MAU/注册数）供用户选择。 |

---

## 5. 交互与多轮上下文

### 5.1 快速确认机制
当出现以下情况时，**禁止猜测**，必须调用 `AskUserQuestion` 工具：
* NLU 置信度 < 0.7。
* 命中“红牌”但未达到直接阻断级别。
* 意图在 `DATA_DEEP` 与 `RESEARCH` 间模糊不清。

### 5.2 上下文延续
* **延续判定**：输入包含“继续、深化、再分析、然后”等短语，且上一轮有明确路由目标。
* **处理逻辑**：**沿用上轮路由目标与参数**，不重新触发全量路由判断，确保任务连贯性。
* **中断重置**：若新输入包含显式的新指令或话题变更，立即重置上下文，重新执行路由逻辑。
