## 一、设计哲学

### 1.1 核心理念

| 理念 | 描述 | 实现方式 |
|------|------|----------|
| **按需加载** | 不预加载全部，按需读取 | Skills 元数据预加载 + 动态读取 SKILL.md |
| **自主决策** | Agent 自主判断，不预设流程 | danalyzer-core 根据需求选择技能组合 |
| **行业适配** | 垂直行业深度定制 | 三层记忆 (Semantic/Episodic/Working) + SQLite 检索 |
| **智能学习** | 从使用中学习，持续优化 | learn/ 系统 (Pattern Detector + Instinct) |

### 1.2 设计原则

```
❌ 不要:
  - 预设固定 Workflow
  - 一次性加载所有数据
  - 依赖外部向量数据库
  
✅ 要:
  - Agent 自主决策
  - 按需加载 + 按需检索
  - 轻量实现 (纯 Python)
  - 行业深度适配
```

## 四、执行流程

### 4.1 完整执行流程

```
用户输入: "分析本月销售趋势，找出增长最快的品类"

Step 1: 需求理解 (danalyzer-core 内嵌)
    │
    ├── 识别行业: 电商 (关键词: 销售)
    ├── 提取指标: 销售额、品类
    ├── 识别时间: 本月
    └── 识别目标: 趋势分析 + 排名
    │
    ▼ (需求模糊?) → 是 → demand-parse
    │
Step 2: 技能决策
    │
    └── 选择技能组合:
        ├── context-retriever (检索行业指标)
        ├── data-query (取数)
        ├── data-analysis (分析)
        ├── visual (可视化)
        └── security (脱敏) [默认嵌入]
    │
    ▼
Step 3: 按需加载 Skills
    │
    └── Read: skills/context-retriever/SKILL.md
        Read: skills/data-query/SKILL.md
        Read: skills/data-analysis/SKILL.md
        Read: skills/visual/SKILL.md
    │
    ▼
Step 4: 执行技能
    │
    ├── context-retriever
    │   └── 检索: sales_amount, category
    ├── data-query
    │   └── SQL: SELECT category, SUM(amount) ...
    ├── data-analysis
    │   └── 计算增长率，排名
    ├── visual
    │   └── 生成趋势图
    └── security
        └── 自动脱敏
    │
    ▼
Step 5: 返回结果
    │
    ├── 数据表格
    ├── 趋势图表
    └── 洞察建议
```
