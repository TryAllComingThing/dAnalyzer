# 数据安全规则

## 核心职责

管控数据输出，确保企业核心数据安全

## 禁止输出类型

### 财务敏感数据
- 未公开的营收、利润数据
- 财务报表详细信息
- 成本结构数据
- 预算/预测数据

### 运营敏感数据
- 核心竞争力数据
- 供应商/客户详细名单
- 库存策略
- 定价策略

### 技术敏感数据
- 数据库连接信息
- API密钥/凭证
- 系统架构图
- 源代码

### 人员敏感数据
- 员工个人信息
- 薪酬福利数据
- 绩效考核数据
- 招聘计划

## 分级管控

| 数据级别 | 描述 | 输出要求 |
|----------|------|----------|
| L4-绝密 | 核心商业机密 | 严格授权，禁止输出 |
| L3-机密 | 重要业务数据 | 部门负责人审批 |
| L2-内部 | 内部使用数据 | 需脱敏处理 |
| L1-公开 | 可公开数据 | 无限制 |

## 审批流程

### L4级别
1. 部门负责人申请
2. 总监审批
3. 安全总监审批
4. CEO审批（可选）

### L3级别
1. 部门负责人申请
2. 总监审批

### L2级别
1. 自动脱敏
2. 记录日志

## 校验方式

- 前置校验：数据分级标识
- 后置校验：输出内容扫描

## 处理方式

- 禁止输出 → 立即终止
- 需审批 → 暂停等待审批
- 脱敏处理 → 自动脱敏后输出
