# 命名规范规则

## 核心职责

统一数据对象的命名规范

## 命名规范

### 表命名规范

```
[用途]_[主题]_[粒度]_[版本]

示例：
dim_user_info        - 维度表-用户信息
fct_order_daily      - 事实表-订单-日
rpt_sales_weekly     - 报表-销售-周
tmp_intermediate     - 临时表-中间结果
```
- 使用小写字母+下划线
- 长度不超过30个字符
- 避免缩写（除非通用）

### 字段命名规范

```
[数据类型]_[业务含义]

示例：
user_id              - 用户ID
order_amount         - 订单金额
create_time          - 创建时间
is_deleted           - 是否删除（布尔型前缀is_）
cnt_                 - 计数字段前缀cnt_
amt_                 - 金额字段前缀amt_
```

### 文件命名规范

```
[业务线]_[报告类型]_[时间范围]_[版本].[格式]

示例：
电商_周报_2026年第17周_v1.md
销售_月报_2026年4月_v1.xlsx
```

### 指标命名规范

```
[业务含义]_[计算方式]_[时间粒度]

示例：
sales_amount_daily   - 销售额-求和-日
order_cnt_monthly    - 订单量-计数-月
user_active_daily    - 活跃用户-去重-日
```

## 禁止命名

- 禁止使用中文
- 禁止使用特殊字符
- 禁止数字开头
- 禁止与保留字冲突

## 校验方式

CLI提醒式校验，仅给出优化建议
