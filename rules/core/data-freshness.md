# 数据时效性规则

## 核心职责

确保数据的时效性满足业务要求

## 时效要求

### 数据更新时效

| 数据类型 | 更新频率 | 最大延迟 | 预警阈值 |
|----------|----------|----------|----------|
| 实时数据 | 实时 | 5分钟 | 10分钟 |
| 日数据 | T+1 | 次日9:00 | 次日12:00 |
| 周数据 | T+1周 | 周一12:00 | 周一18:00 |
| 月数据 | T+1月 | 次月3日 | 次月5日 |

### 数据有效期

| 数据类型 | 有效期限 | 说明 |
|----------|----------|------|
| 实时数据 | 5分钟 | 需实时刷新 |
| 日数据 | 3天 | 3天后标记为历史 |
| 周数据 | 2周 | 2周后标记为历史 |
| 月数据 | 3个月 | 3月后标记为历史 |

### 数据完整性

| 数据类型 | 完整性要求 |
|----------|------------|
| 日数据 | 100% |
| 周数据 | 100% |
| 月数据 | 100% |

## 校验方式

CLI前置校验，检查数据更新时间

## 处理方式

- 超时未更新 → 预警通知
- 完整性不足 → 预警+日志
- 历史数据使用 → 警告
