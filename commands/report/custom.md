# /report custom - 自定义报告

## 功能说明

生成灵活的自定义报告:
- 自选指标
- 自定义维度
- 自定义时间范围
- 自定义模板

## 使用方式

```
/report custom <报告配置>
```

## 配置方式

### 方式1: JSON 配置

```bash
/report custom {
  "name": "月度销售报告",
  "metrics": ["销售额", "订单数", "客单价"],
  "dimensions": ["日期", "地区", "品类"],
  "time_range": "2026-03",
  "filters": {"地区": "华东"},
  "template": "detailed"
}
```

### 方式2: YAML 配置

```bash
/report custom name: 月度销售报告
  metrics: 销售额,订单数,客单价
  dimensions: 日期,地区
  time_range: 2026-03
```

### 方式3: 模板继承

```bash
# 基于模板修改
/report custom --template=sales_monthly --override metrics:销售额,毛利率
```

## 使用示例

### 销售分析报告

```bash
/report custom name: 华东区Q1销售报告
  metrics: 销售额,订单数,毛利率
  dimensions: 月份,省份,城市
  time_range: 2026-Q1
  filters: 地区:华东
```

### 用户分析报告

```bash
/report custom name: 用户留存报告
  metrics: 新增用户,活跃用户,留存率
  dimensions: 渠道,用户类型
  time_range: last_30d
```

### 财务报告

```bash
/report custom name: 利润分析报告
  metrics: 销售额,成本,毛利,毛利率
  dimensions: 品类,部门
  time_range: 2026-01~2026-03
```

## 配置项说明

| 配置项 | 必填 | 说明 |
|--------|------|------|
| name | 是 | 报告名称 |
| metrics | 是 | 指标列表 |
| dimensions | 否 | 维度列表 |
| time_range | 是 | 时间范围 |
| filters | 否 | 筛选条件 |
| template | 否 | 模板名称 |

## 时间范围格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 日期 | 2026-04-26 | 单日 |
| 月份 | 2026-03 | 自然月 |
| 季度 | 2026-Q1 | 季度 |
| 年份 | 2026 | 整年 |
| 范围 | 2026-01~2026-03 | 日期范围 |
| 相对 | last_7d | 相对日期 |

## 保存模板

```bash
# 保存为模板
/report custom --save my_template

# 使用保存的模板
/report custom --template=my_template
```
