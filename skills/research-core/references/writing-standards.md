# 写作标准

## 叙事驱动

用流畅的散文写作。每项发现讲述一个故事：开头（背景）→ 中间（证据）→ 结尾（启示）。

```
✅ "受消费者需求增长和监管利好推动，2023 年全球市场规模达到 240 亿美元 [1]。
   这一增长主要由亚太地区引领，该地区贡献了总增量的 45% [2]。然而，监管碎片化
   仍然是跨国企业面临的主要障碍 [3]。"

❌ "• 市场规模：240 亿美元  • 增长地区：亚太  • 主要障碍：监管"
```

## 要点政策

- 仅用于列举（产品名称、公司名录、步骤清单）
- 禁止将要点作为主体内容呈现——要点会割裂论证逻辑
- 每个 `##` 标题下 ≥ 3 段完整散文，要点内容占比 < 20%

## 「中间淹没」预防

- 关键发现放在每节的开头或结尾，不埋在中间段落
- 每节结构：摘要句 → 详细论证 → 收束结论
- 使用明确的标题引导读者注意力

## 抗疲劳质量检查（逐节执行）

在确认任一章节完成前，核实：

- 段落数：每个 `##` 标题下 ≥ 3 段
- 散文优先：要点式内容 < 20%
- 无占位符：无"内容待补充""TBD""TODO"
- 证据充分：具体数据/统计/引述，非模糊陈述
- 引用密度：重要论断首次提出时即附 [N]

**如果任何一项检查失败 → 重新生成该节。**
