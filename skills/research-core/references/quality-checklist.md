# 质量保证

## 验证清单（最终输出前逐项检查，不通过不输出）

- [ ] 摘要 50-250 字，覆盖核心发现 + 关键结论
- [ ] 必需章节完整（执行摘要 → 正文 → 结论与建议 → 局限性 → 附录）
- [ ] 引用格式统一为 `[N]`，参考文献与正文引文一一对应
- [ ] 无占位符（TBD、TODO、「内容待补充」）
- [ ] 字数达到深度级别下限（L1: 2000+, L2: 6000+, L3: 10000+）
- [ ] 来源数达到深度级别下限（L1: ≥10, L2: ≥20, L3: ≥30）
- [ ] 核心论点有足够独立来源（L1: ≥2, L2: ≥2 交叉验证, L3: ≥3）
- [ ] 每个 `##` 标题下 ≥ 3 段散文，要点占比 < 20%
- [ ] 所有统计数据标注了年份/时间范围
- [ ] 无编造的事实（每个 `[N]` 可追溯到具体 URL）
- [ ] 「局限性」章节包含：信息缺口、不确定性说明、方法论局限
- [ ] 附录包含完整参考文献（URL 或文献标识符）+ 方法论说明

## 验证失败处理

1. 格式/链接问题 → 自动修复
2. 内容问题 → 重新生成受影响章节
3. 两次修复失败 → 标注问题，向用户说明，交付当前版本
