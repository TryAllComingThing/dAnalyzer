# 合规检查清单 (Compliance Checklist)

## 敏感数据类型检测

### P0 - 绝对禁止输出
| 类型 | 正则/规则 | 严重程度 |
|------|-----------|----------|
| 身份证号 | `\d{17}[\dXx]` | CRITICAL |
| 银行卡号 | `\d{16,19}` | CRITICAL |
| 密码/密钥 | `password\|secret\|key\|token` | CRITICAL |

### P1 - 必须脱敏
| 类型 | 正则/规则 | 脱敏规则 |
|------|-----------|----------|
| 手机号 | `1[3-9]\d{9}` | 138****1234 |
| 邮箱 | `[\w.-]+@[\w.-]+` | a***@example.com |
| 姓名 | 常见姓氏+名 | 张* |
| 地址 | 含省市区街道 | 北京市*** |

### P2 - 建议脱敏
| 类型 | 规则 |
|------|------|
| IP地址 | 最后一段脱敏 |
| MAC地址 | 全部脱敏 |
| 账号ID | 部分脱敏 |

## 脱敏规则

```yaml
手机号: "前3位+****+后4位"
身份证: "前6位+********+后4位"
银行卡: "**** **** **** **** +后4位"
邮箱: "首字符***@域名"
姓名: "姓+*"
地址: "省***市***区/县***"
```

## 合规验证项

### 输出前必须检查
- [ ] 无P0级敏感数据
- [ ] P1级数据已脱敏
- [ ] 数据来源合法
- [ ] 无违规内容

### 审计日志必须记录
- 操作时间
- 操作人
- 数据类型
- 脱敏方式
- 处理结果

## 响应级别

| 级别 | 动作 |
|------|------|
| CRITICAL | 立即终止，提示用户 |
| HIGH | 脱敏后继续，警告用户 |
| MEDIUM | 脱敏后继续，提醒用户 |
| LOW | 记录日志，继续执行 |
