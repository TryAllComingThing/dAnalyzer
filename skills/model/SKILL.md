# 数据建模技能 (Model)

> **使用说明**：当前阶段模型执行依托 Claude 通用知识推理，无独立 Python 模型实现。子技能文件为方法参考文档（概念定义 + 执行流程），非可调用模块。

## 概述

数据建模方法参考技能，覆盖 8 种分析模型：归因分析、聚类分析、队列分析、相关性分析、预测模型、漏斗模型、RFM 模型、趋势分析。采用 Tool Wrapper + Multi-step 模式 — 自动推荐模型 → 用户确认 → 执行 → 结果校验 → 输出。各模型详细方法见 references/ 子技能文件。

## 何时使用

- **触发:** 用户需要归因分析、用户分群或聚类
- **触发:** 用户需要 RFM 评分、趋势预测或漏斗/转化分析
- **触发:** 用户需要队列/留存分析或相关性建模
- **不要用于:** 简单描述性统计（使用 data-analysis）
- **不要用于:** 数据查询或清洗（使用 data-query 或 data-clean）
- **不要用于:** 模型结果可视化（使用 visual）

---

## 核心步骤

1. **分析输入数据** — 检查数据特征（数据量、字段类型、业务场景）→ 详见「执行流程 > 步骤 1」
2. **模型推荐** — 基于数据特征和用户需求自动推荐最佳模型及理由 → 详见「执行流程 > 步骤 1」
3. **用户确认** — 展示推荐模型，等待用户确认或调整 → 详见「执行流程 > 步骤 2」
4. **执行模型** — Read 对应 references/ 子技能文件，按方法执行 → 详见「执行流程 > 步骤 3」和「子技能参考文件」
5. **结果校验** — 验证模型输出质量，检查指标合理性；异常则提示重新选择 → 详见「执行流程 > 步骤 4」
6. **输出结果** — 模型结果数据 + 分析报告 + 建议 → 详见「执行流程 > 步骤 5」

## 方法类型

1. **归因分析** - 多渠道归因、贡献度分析
2. **聚类分析** - 用户分群、行为分组
3. **队列分析** - 用户留存、同期群分析
4. **相关性分析** - 变量关系、影响因素
5. **预测模型** - 趋势预测、销量预测
6. **漏斗模型** - 转化分析、流失分析
7. **RFM模型** - 用户价值分层
8. **趋势分析** - 走势预测、季节性

## 使用场景

### 场景1: 用户分群
```
用户: 按用户行为分群
→ 调用 model 技能 → [校验: 模型选择] → 用户确认 → 聚类分析
→ 输出用户分群结果
```

### 场景2: 营销归因
```
用户: 分析各渠道贡献度
→ 调用 model 技能 → [校验: 模型选择] → 归因分析
→ 输出各渠道贡献占比
```

## 执行流程 (Tool Wrapper + Multi-step 模式)

> 对应核心步骤第 1-6 步

```
[开始] → [校验: 模型选择器] → [用户确认] → [执行模型] → [校验: 结果校验] → [输出]
                          ↓ 不确认
                      [重新选择]
```

### 详细步骤

1. **[校验] 模型选择器 (Auto-Selection)**
   - 分析输入数据特征（数据量、字段类型、业务场景）
   - 根据 `references/model-selection-rules.md` 规则推荐最佳模型
   - 输出推荐模型列表及理由

2. **[MULTI-STEP] 用户确认**
   - "推荐使用 [模型名称]，原因：..."
   - 等待用户确认后执行
   - 用户可选择其他模型或调整参数

3. **执行选定的模型**
   - 归因分析 → references/attribution.md
   - 聚类分析 → references/clustering.md
   - 预测模型 → references/forecasting.md
   - 等等

4. **[校验] 结果校验**
   - 验证模型输出质量
   - 检查指标是否合理
   - **如果结果异常：提示重新选择模型**

5. **输出结果**
   - 模型结果数据
   - 分析报告
   - 建议

## 常见借口与纠正

| 借口 | 现实 |
|--------|---------|
| "直接用最复杂的模型，结果一定最好" | 模型复杂度应与数据量和业务需求匹配；简单数据用复杂模型反而过拟合 |
| "用户说了要做聚类，我就选聚类" | 用户指定的模型不一定最优；校验器应给出推荐并说明理由 |
| "模型输出看起来很合理，不需要校验" | 所有模型输出必须经过校验步骤；异常结果静默输出导致错误决策 |

## 红线

- **数据量不足:** 数据量对所选模型过小（如 <100 行做聚类）— 推荐更简单方法或更多数据
- **输出违反业务逻辑:** 模型输出严重违背业务常识 — 重新检查输入和假设
- **确认步骤被绕过:** 用户要求跳过模型确认直接执行 — 强制执行校验关卡
- **校验连续失败:** 结果校验连续 2 次失败 — 标记并重新评估模型选择

## 验证

1. 验证模型选择有依据：确认所选模型有数据特征支撑（非随机或硬编码）
2. 验证用户已确认：确认用户批准了模型选择（或默认推荐已记录）
3. 验证参考文件已加载：确认对应 references/ 子技能文件已被 Read
4. 验证结果已校验：确认模型输出通过了校验步骤
5. 验证输出完整：确认交付物包含模型结果、分析报告和建议
