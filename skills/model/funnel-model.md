# 漏斗分析技能

## 核心功能

分析用户从入口到转化的各阶段转化率

## 模型定义

### 漏斗阶段

电商漏斗示例：
1. **浏览（View）**：用户访问商品页/首页
2. **加购（Cart）**：用户将商品加入购物车
3. **下单（Order）**：用户提交订单
4. **付款（Payment）**：用户完成支付
5. **复购（Repurchase）**：用户再次购买

自定义漏斗：
- 注册流程：注册→实名→绑卡→首次投资
- 营销活动：曝光→点击→领取→使用

## 计算公式

```
阶段转化率 = 当前阶段用户数 / 上一阶段用户数
总体转化率 = 最终阶段用户数 / 第一阶段用户数
流失率 = 1 - 阶段转化率
流失用户数 = 上一阶段用户数 - 当前阶段用户数
```

## 分析指标

| 指标 | 说明 |
|------|------|
| 阶段用户数 | 每个阶段的独立用户数 |
| 阶段转化率 | 当前/上一阶段用户数 |
| 总体转化率 | 最终/第一阶段用户数 |
| 流失用户数 | 各阶段流失的用户数 |
| 流失率 | 流失用户数/上一阶段 |
| 平均转化时间 | 各阶段平均耗时 |

## 执行流程

1. 定义漏斗阶段和事件
2. 提取各阶段用户数据
3. 按用户ID关联各阶段
4. 计算转化率和流失率
5. 识别流失严重的阶段
6. 生成优化建议

## 输出结果

- 漏斗阶段数据表
- 转化率统计表
- 流失用户明细
- 优化建议报告
