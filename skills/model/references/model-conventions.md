# 数据建模规范 (Data Modeling Conventions)

## 模型选择指南

| 分析目标 | 推荐模型 | 适用场景 |
|----------|----------|----------|
| 用户分层 | RFM模型 | 用户价值评估 |
| 转化分析 | 漏斗模型 | 用户行为路径 |
| 归因分析 | 归因模型 | 营销效果评估 |
| 趋势预测 | 预测模型 | 销量/流量预测 |
| 用户分群 | 聚类模型 | 市场细分 |
| 相关性 | 相关分析 | 指标关系 |

## RFM 模型参数

```yaml
R (Recency): 最近消费距今天数
F (Frequency): 统计周期内消费次数
M (Monetary): 统计周期内消费金额

分档阈值（默认值，可调整）:
  R: [30, 90, 180, ∞]  # 天数越小分越高
  F: [1, 3, 6, ∞]      # 次数越大分越高
  M: [100, 500, 2000, ∞]  # 金额越大分越高
```

## 漏斗模型参数

```yaml
阶段定义:
  - 浏览商品
  - 加入购物车
  - 提交订单
  - 付款成功

分析维度:
  - 转化率 (各阶段/前一阶段)
  - 流失率 (1 - 转化率)
  - 平均转化时间
```

## 模型输出规范

- 模型参数配置
- 模型评估指标 (准确率、AUC等)
- 业务解释和建议

## 注意事项

1. 建模前必须进行数据清洗
2. 异常值需单独处理
3. 模型结果需业务验证
4. 定期更新模型参数
