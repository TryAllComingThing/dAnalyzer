# 漏斗定义咨询阶段

## 目的
在执行漏斗分析前，必须明确漏斗的阶段定义和业务规则。

## 必须确认的问题

### 1. 漏斗场景
```
□ 电商购物流程
□ 用户注册流程
□ 付费转化流程
□ 自定义流程: _______
```

### 2. 漏斗阶段（至少2个阶段）
```
阶段1: _________   定义: _________
阶段2: _________   定义: _________
阶段3: _________   定义: _________
阶段4: _________   定义: _________
阶段5: _________   定义: _________
```

### 3. 时间窗口
- 用户行为时间范围: 从 _______ 到 _______
- 阶段间隔时间限制（可选）: _________

### 4. 过滤条件（可选）
- 用户类型: _________
- 商品类目: _________
- 地域: _________
- 其他: _________

### 5. 成功标准
- 转化率目标: _________
- 参考基准: _________

## 咨询模板

```
开始分析前，请确认以下信息：

1. 您的业务场景是？（电商/注册/付费/其他）
2. 您想分析的漏斗阶段有哪些？请按顺序列出
3. 分析的时间范围是什么？
4. 是否有需要过滤的用户或条件？
5. 您期望达到的转化率目标是多少？
```

## 规则

- **未确认以上信息前，禁止开始数据分析**
- 如果信息不完整，提示用户提供
- 确认后记录到执行上下文
