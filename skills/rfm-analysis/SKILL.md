# RFM Analysis Skill

## 核心职责

- 计算用户 RFM 指标 (Recency, Frequency, Monetary)
- 用户分群和价值分层
- 识别流失风险用户
- 生成用户营销建议

## 适用场景

- 用户价值评估
- 客户生命周期管理
- 精准营销用户选择
- 用户流失预警

## RFM 概念

| 维度 | 说明 | 业务含义 |
|------|------|----------|
| **R (Recency)** | 最近一次消费时间 | 流失可能性，越近越好 |
| **F (Frequency)** | 消费频率 | 忠诚度，越高越好 |
| **M (Monetary)** | 消费金额 | 价值贡献，越高越好 |

## 输入要求

| 字段 | 必填 | 说明 |
|------|------|------|
| analysis_type | 是 | 分析类型 (basic/advanced) |
| reference_date | 是 | 参考日期 |
| time_window | 是 | 分析时间窗口 (月数) |
| rfm_scores | 否 | RFM 分段数量 (默认 5) |

## 执行逻辑

### 1. 计算 RFM 指标

```
R (最近消费):
- 计算每个用户到最后一次消费的天数
- 天数越少，R 值越高

F (消费频率):
- 统计时间窗口内的订单次数
- 次数越多，F 值越高

M (消费金额):
- 统计时间窗口内的消费总额
- 金额越高，M 值越高
```

### 2. RFM 分值计算

```
分值计算 (5分制):
- R: 按天数分段，1-5 分 (天数少=高)
- F: 按次数分段，1-5 分 (次数多=高)
- M: 按金额分段，1-5 分 (金额高=高)
```

### 3. 用户分群

```
8类用户模型:
┌─────────┬─────────┬─────────┬─────────┐
│ R值     │ F值     │ M值     │ 用户类型  │
├─────────┼─────────┼─────────┼─────────┤
│ 高      │ 高      │ 高      │ 核心客户  │
│ 高      │ 高      │ 低      │ 潜力客户  │
│ 高      │ 低      │ 高      │ 价值客户  │
│ 高      │ 低      │ 低      │ 新客户    │
│ 低      │ 高      │ 高      │ 重要挽留  │
│ 低      │ 高      │ 低      │ 流失预警  │
│ 低      │ 低      │ 高      │ 高消费潜逃│
│ 低      │ 低      │ 低      │ 流失客户  │
└─────────┴─────────┴─────────┴─────────┘
```

## 注意事项

1. 时间窗口选择要合理 (建议3-6个月)
2. RFM 分数分段要考虑行业特点
3. 异常值需要特殊处理
4. 结合行业特点调整分群规则
