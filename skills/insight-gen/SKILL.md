# Insight Generation — 洞察生成

## When to Activate

- 分析完成后需要提炼业务结论
- 用户要求"有什么发现"、"给出建议"
- 报告需要洞察摘要章节

---

## 洞察提炼规则

分析结果到手后，按以下框架提炼洞察：

### 1. 识别信号

| 信号 | 触发条件 | 示例输出 |
|------|---------|---------|
| 趋势信号 | 连续 3 期同向变化，或环比 > 10% | "销售额连续 3 周增长，周均 +12%" |
| 异常信号 | 超出 IQR/3σ 范围，或业务合理范围 | "周二订单量骤降 75%，疑为系统延迟" |
| 对比信号 | 组间差异 > 20%，Cohen's d > 0.5 | "华东区域 ARPU 显著高于其他区域 (d=0.7)" |
| 结构信号 | 单一维度占比 > 40% | "电子产品贡献 45% 营收，集中度偏高" |

### 2. 评估可信度

- **高**: 样本充足 (>1000)，趋势一致，统计显著
- **中**: 样本一般 (100-1000)，趋势模糊，波动大
- **低**: 样本不足 (<100)，单一数据点，需更多数据验证

### 3. 生成业务建议

每个洞察必须附带可执行的业务建议：

```
数据发现 → 业务含义 → 建议行动
"退款率上升 5%" → "客服可能无法满足新增用户需求" → "建议加派人手并优化自助退款流程"
```

### 4. 优先级排序

| 优先级 | 条件 |
|--------|------|
| **P0-紧急** | 核心指标异常偏离 > 50%，需立即处理 |
| **P1-重要** | 影响 > 20% 用户/营收，24h 内处理 |
| **P2-关注** | 趋势信号，本周内跟进 |
| **P3-信息** | 一般性发现，记录备查 |

---

## 输出格式

```markdown
## 洞察摘要

### 核心发现
1. **[信号类型]** 发现描述 — 数据支撑 — 可信度
2. ...

### 业务建议
1. 建议行动 — 预期效果 — 优先级

### 关注事项
- ⚠️ 需关注的风险/异常
- ✅ 积极信号
```
