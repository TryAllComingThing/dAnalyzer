# Report — 报告生成技能

## 概述

结构化报告生成技能，支持日报、周报、月报、对比报告、临时报告五种类型。从上游分析结果中提取数据，按模板填充占位符，输出 Markdown 正文 + CSV 数据附件。所有报告输出前强制经 security 脱敏。

## 何时使用

- **触发:** 用户要求生成报告：日报、周报、月报、对比报告或临时报告
- **触发:** 分析已完成，需要带叙述的格式化输出
- **不要用于:** 纯图表看板（使用 dashboard）
- **不要用于:** 无分析叙述的原始数据导出（使用 data-query + 导出）
- **不要用于:** 分析尚未完成时生成报告

---

## 核心步骤

1. **确定报告类型** — 日报/周报/月报/对比报告/临时报告，匹配对应模板段落 → 详见「报告类型」
2. **加载模板** — `Read skills/report/assets/report-template.md`，获取精确章节结构
3. **数据映射** — 按「数据来源映射」表将上游数据列映射到占位符
4. **填充内容** — 数值占位符填数据，自由文本占位符填叙述，图表占位符填图片语法 → 详见「报告格式规范」
5. **写入附件** — 通过 CSVConnector 写入同名 .csv 数据附件
6. **安全脱敏** — 经 security 扫描后输出 .md + .csv

## 报告类型

> 对应 核心步骤 第 1 步

| 类型 | 结构概要 | 模板 |
|------|----------|------|
| **日报** | 今日概览 → 实时数据 → 核心指标趋势 → 异常预警 → 关注事项 | `assets/report-template.md` |
| **周报** | 本周摘要 → 核心指标（环比）→ 维度拆解 → TOP 榜单 → 异常说明 → 下周关注 | `assets/report-template.md` |
| **月报** | 月度摘要 → 指标趋势（同比+环比）→ 渠道/商品/用户深度分析 → 异常检测 → 下月展望 | `assets/report-template.md` |
| **对比报告** | 对比概览 → 差异明细 → 可视化对比 → 原因分析 → 结论建议 | `assets/report-template.md` |
| **临时报告** | 自由结构，按用户需求定制 | 无固定模板，参见 `references/comparison-report.md` 的三种对比类型 |

## 核心指标模板

报告自动匹配行业指标模板。以下为默认模板，行业特定指标优先使用 `context-card` 中的定义。

### 电商通用指标

| 指标 | 计算公式 |
|------|---------|
| GMV | `SUM(actual_amount)` |
| 订单量 | `COUNT(DISTINCT order_id)` |
| 客单价 | `GMV / 订单量` |
| 转化率 | `下单用户数 / 访问用户数` |
| 退款率 | `退款订单数 / 总订单数` |
| ARPU | `GMV / 用户数` |

### 制造业指标

| 指标 | 计算公式 |
|------|---------|
| 产能利用率 | `实际产出 / 设计产能 × 100%` |
| 不良品率 | `不良品数 / 总产量 × 100%` |
| 设备综合效率 (OEE) | `可用率 × 性能率 × 质量率` |
| 计划停机时间 | `SUM(scheduled_downtime_hours)` |
| 单位能耗 | `总能耗 / 总产量` |

### 物流指标

| 指标 | 计算公式 |
|------|---------|
| 准时交付率 | `准时交付单数 / 总单数 × 100%` |
| 平均转运天数 | `AVG(签收日期 - 发货日期)` |
| 单均成本 | `总成本 / 总单数` |
| 破损率 | `破损件数 / 总件数 × 100%` |
| 在途库存周转 | `SUM(in_transit_qty) / 日均出库量` |

### 环比/同比

`环比 = (本期-上期)/上期 × 100%`  `同比 = (本期-去年同期)/去年同期 × 100%`

### 变化判定

| 阈值 | 标记 | 行动 |
|------|------|------|
| ≤5% | 持平 | 正常波动 |
| 5-10% | 小幅 ↑/↓ | 关注 |
| 10-20% | 显著 ↑/↓ | 分析原因 |
| >20% | 大幅 ↑/↓ | 必须解释 |
| >50% | 异常 ↑/↓ | 核验数据 |

## 报告格式规范

> 对应 核心步骤 第 4 步

- 标题包含时间范围和主题，摘要 3-5 句先结论后数据
- 数字千分位分隔，保留 1-2 位小数
- 每个图表有标题和轴标签（含单位），配色统一使用 skills/visual/references/chart-standard.md

### 报告导出

```
输出格式: Markdown
数据附件: CSV（与报告正文同名，同目录）
命名规范: {报告类型}_{日期}.md / .csv
```

## 常见借口与纠正

| 借口 | 现实 |
|--------|---------|
| "数据都在，我直接写 Markdown 更快" | 跳过模板会导致章节结构不一致，多份报告风格割裂 |
| "变化不到 5%，不用在报告里提" | 持平也是信息；报告中应标注「正常波动」而非省略 |
| "对比维度不够，意思到了就行" | 对比报告必须有差异明细表 + 原因分析，仅表格无结论价值有限 |

## 红线

- **报告含未验证数据:** 报告填充了未经 data-analysis 或 model 验证的数据 — 报告可能包含错误
- **极端变化未标注:** 核心指标变化超 50% 但未包含数据异常警告 — 必须向用户标注
- **安全扫描被绕过:** 报告输出未通过 security 脱敏 — PII 可能泄露
- **占位符残留:** 模板中存在未填充的 `{{ }}` 占位符 — 报告不完整

## 验证

1. 验证报告类型：确认所选类型（日报/周报/月报/对比/临时）匹配用户需求
2. 验证模板已加载：确认 `report-template.md` 在填充内容前已被 Read
3. 验证无占位符残留：确认输出中无 `{{ }}` 模式残留
4. 验证数字格式：确认千分位分隔和 1-2 位小数一致应用
5. 验证 CSV 附件：确认同名 .csv 数据文件与 .md 报告并列存在
6. 验证安全扫描：确认最终输出在交付前通过 `security_scan.py`
