# 数据分析技能 (Data Analysis)

> **使用说明**：本 Skill 定义标准统计分析方法与公式。实际分析执行委托给 industry-specific analyzer（如 industry_analyzers.py），它们可能根据业务场景使用简化聚合。本 Skill 的公式作为质量参考和异常检测指南。

## 概述

统计分析参考技能，定义描述性统计、趋势分析、对比分析、相关性分析和分布分析的标准方法与公式。覆盖异常值检测（IQR/Z-score/MAD/DBSCAN）、效应量计算（Cohen's d）、趋势显著性检验（Mann-Kendall），并规定洞察提炼五步规则（数据特征 → 变化方向 → 显著差异 → 业务含义 → 建议行动）。

## 何时使用

- **触发:** 用户需要从数据中提取统计特征或洞察
- **触发:** 用户需要趋势分析、对比分析、相关性分析或分布分析
- **触发:** 用户需要假设检验或统计显著性评估
- **触发:** 用户需要异常检测或离群值分析
- **不要用于:** 原始数据查询（使用 data-query）
- **不要用于:** 数据清洗（使用 data-clean）
- **不要用于:** RFM、聚类、预测等高级建模（使用 model）
- **不要用于:** 无统计分析的纯可视化（使用 visual）

## 核心步骤

1. **确定分析类型** — 根据用户需求选择：描述性统计/趋势分析/对比分析/相关性分析/分布分析 → 详见「分析类型」
2. **执行统计计算** — 按本 Skill 定义的公式计算相应指标 → 详见「分析类型」各子节
3. **异常值检测** — 选择合适方法（IQR/Z-score/MAD/DBSCAN），分类判断处理方式 → 详见「分析类型 > 1. 描述性统计 > 异常值判定规则」
4. **效应量评估** — 差异/变化是否具有实际意义（Cohen's d、百分比差异）→ 详见「分析类型 > 3. 对比分析 > 效应量指标」
5. **洞察提炼** — 按五步规则输出：数据特征 → 变化方向 → 显著差异 → 业务含义 → 建议行动 → 详见「洞察提炼规则」

## 分析类型

> 对应 核心步骤 第 1-4 步

### 1. 描述性统计

| 指标 | 计算方法 | 适用场景 |
|------|---------|---------|
| 均值 (Mean) | `SUM(values) / COUNT(values)` | 中心趋势，正态分布数据 |
| 中位数 (Median) | 排序后取中间值 | 偏态分布，有异常值时的中心趋势 |
| 标准差 (Std) | `SQRT(SUM((x-μ)²)/N)` | 离散程度 |
| 四分位数 (Q1/Q3) | 排序后 25%/75% 位置 | 数据分布范围，箱线图基础 |
| IQR | `Q3 - Q1` | 异常值检测阈值 |
| 偏度 (Skewness) | 分布不对称性 | 判断数据是否正态 |
| 峰度 (Kurtosis) | 分布尾部厚度 | 识别厚尾/薄尾分布 |

**异常值判定规则**:

| 方法 | 公式/逻辑 | 适用场景 | 优势 |
|------|----------|---------|------|
| **IQR 法** | `value < Q1 - 1.5×IQR` 或 `value > Q3 + 1.5×IQR` | 偏态分布 | 无分布假设，鲁棒 |
| **Z-score 法** | `|z| > 3`（z = (x-μ)/σ） | 正态分布 | 简单直观 |
| **MAD 法** | `|x - median| / MAD > 3`（MAD = median(|x - median|)） | 含异常值样本 | 比 Z-score 更鲁棒，崩溃点 50% |
| **DBSCAN** | 基于密度的空间聚类，低密度点 = 异常 | 多维异常检测 | 不需要预设异常数量，可发现任意形状 |
| **时间序列异常** | 移动平均残差法：`|actual - MA(n)| > k × σ` | 时间序列数据 | 考虑时间上下文 |

**异常值处理决策**:
```
检测到异常值 → 分类判断:
├── 数据错误（负数价格、未来日期）→ 修正或剔除
├── 真实极端值（大额订单、黑天鹅事件）→ 保留并标注
└── 不确定 → 保留，记录在分析 limitations 中
```

### 2. 趋势分析

```
分析维度:
├── 时间趋势: 按日/周/月聚合，观察方向
├── 环比: (本期 - 上期) / 上期 × 100%
├── 同比: (本期 - 去年同期) / 去年同期 × 100%
├── 移动平均: 平滑短期波动，识别长期趋势
│   - SMA(n): 简单移动平均，窗口 n
│   - EMA(n): 指数移动平均，权重递减
└── 趋势显著性: Mann-Kendall 检验（非参数）
```

**趋势判定标准**:
- 连续 3 期同向变化 → 趋势信号
- 环比变化 > 10% → 显著变化（需结合业务判断）
- 明显拐点 → 标注可能原因

### 3. 对比分析

```
对比维度:
├── 时间对比: 环比/同比
├── 分组对比: 按维度分组（地区/产品/渠道）
├── 基准对比: 与目标值/行业均值对比
└── 排名分析: Top N / Bottom N

效应量指标:
├── Cohen's d: 两组均值差异 (0.2小/0.5中/0.8大)
├── 百分比差异: (A-B)/B × 100%
└── 相对风险: 发生率比值
```

### 4. 相关性分析

| 方法 | 适用数据类型 | 阈值 |
|------|-------------|------|
| Pearson r | 连续-连续（线性） | \|r\|>0.7 强, >0.4 中, <0.2 弱 |
| Spearman ρ | 有序/非正态 | 同上 |
| Cramer's V | 分类-分类 | >0.3 强, >0.1 中 |
| 互信息 (MI) | 非线性关系 | 相对值比较 |

**注意**: 相关性 ≠ 因果性。显著相关 (p<0.05) 只说明统计上非零。

### 5. 分布分析

```
分析方法:
├── 直方图: 观察分布形态
├── Q-Q 图: 正态性视觉检验
├── Shapiro-Wilk: 正态性统计检验 (p<0.05 → 非正态)
├── 核密度估计 (KDE): 平滑分布曲线
└── 多峰检测: 子群体识别
```

## 方法选择指南

> 根据数据特征选择合适的统计方法，避免误用。

### 数据特征 → 方法映射

| 数据特征 | 推荐方法 | 避免使用 |
|----------|---------|---------|
| 正态分布 + 连续变量 | 均值、标准差、Pearson r、t 检验 | — |
| 偏态分布 + 连续变量 | 中位数、IQR、Spearman ρ、Mann-Whitney U | 均值（有误导性） |
| 有序分类变量 | Spearman ρ、Kendall τ | Pearson r |
| 无序分类变量 | Cramer's V、卡方检验 | Pearson r |
| 小样本 (n<30) | 非参数检验（Mann-Whitney、Wilcoxon） | t 检验、ANOVA |
| 大样本 + 多维 | PCA、聚类、DBSCAN | 单变量方法 |
| 时间序列 | Mann-Kendall、移动平均、ARIMA | 忽略自相关的普通回归 |
| 非线性关系 | 互信息 (MI)、Spearman ρ、GAM | Pearson r |

### 参数 vs 非参数选择

| 条件 | 参数检验 | 非参数检验 |
|------|---------|-----------|
| 数据符合正态分布 | ✅ | 可用但功效略低 |
| 数据严重偏态 | ❌ | ✅ |
| 样本量 < 30 | ❌ 不可靠 | ✅ |
| 含极端异常值 | ❌ 均值被拉偏 | ✅ 基于秩次 |
| 方差齐性 | ✅ | ✅ |

常用非参数替代：t 检验 → Mann-Whitney U、配对 t → Wilcoxon、ANOVA → Kruskal-Wallis、Pearson r → Spearman ρ

### 显著性阈值

| 场景 | 阈值 | 说明 |
|------|------|------|
| 常规分析 | p < 0.05 | 默认标准 |
| 多重比较 | p < 0.05/n（Bonferroni）或 FDR < 0.05 | 防止假阳性膨胀 |
| 探索性分析 | p < 0.10 | 放宽标准，标注为"初步发现" |
| 关键业务决策 | p < 0.01 | 更严格，降低决策风险 |

**注意**：p 值仅说明统计显著性，不说明实际意义。必须结合效应量（Cohen's d、百分比差异）综合判断。

## 洞察提炼规则

分析完成后按深度要求输出文本，包含：
- **L1 快速摘要**（默认）：数据特征（中心/离散/分布）+ 主要差异或变化方向。
- **L2 标准解读**：特征 + 方向 + 显著差异（含效应量）+ 业务含义。
- **L3 深度报告**：L2 + 异常值说明 + 方法选择理由 + 局限性 + 建议下一步行动。

输出格式示例：
> **数据特征**：销售额分布右偏，中位数120万，IQR 80-200万。发现3个极端值（>500万）。
> **变化方向**：环比增长15%（p=0.02，Cohen's d=0.6），趋势Mann-Kendall p=0.03，显著上升。
> **差异**：A组与B组差异不显著（校正后 p=0.18），但C组显著高于其他（η²=0.20）。
> **业务含义**：整体增长由C组驱动，A/B组需重点关注。极端值可能对应大宗交易，建议保留分层分析。
> **建议**：分析C组成功因素，对A/B组进行根因诊断。

## 常见借口与纠正

| 借口 | 现实 |
|--------|---------|
| "均值已经能说明问题了" | 偏态分布下均值有误导性；中位数和分布形态同样重要 |
| "相关性高就是因果" | 相关性 ≠ 因果性；显著相关 (p<0.05) 只说明统计上非零 |
| "异常值自动删除即可" | 异常值可能是关键业务信号；先分类再决定，不确定时保留标注 |
| "样本量不够，参数检验也能用" | n<30 时参数检验不可靠，必须使用非参数替代 |

## 红线（不可违反）
- 小样本 (n<30) 禁止使用参数检验，除非用户指定且记录风险。
- 多重比较必须校正，不得直接使用原始 p 值画星号。
- 异常值不得直接删除，必须先分类并记录。
- 相关分析前必须绘制散点图等视觉检查，不得仅凭数值下结论。
- 单次分析中剔除异常值比例 >5% 需中断并报告用户。

## 验证
技能执行完成后，自动进行以下自我检查：
1. 分析类型与问题匹配。
2. 选定检验方法已记录，且与数据分布/样本量兼容。
3. 异常值处理已记录分类和比例。
4. 效应量与 p 值同时呈现。
5. 洞察包含了业务含义和局限性。
