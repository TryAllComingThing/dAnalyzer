# 数据分析技能 (Data Analysis)

## When to Activate

- 用户需要从数据中提取统计特征或洞察
- 用户需要趋势分析、对比分析、相关性分析
- 用户需要假设检验或显著性判断
- 用户需要异常检测或分布分析

## 分析类型

### 1. 描述性统计

| 指标 | 计算方法 | 适用场景 |
|------|---------|---------|
| 均值 (Mean) | `SUM(values) / COUNT(values)` | 中心趋势，正态分布数据 |
| 中位数 (Median) | 排序后取中间值 | 偏态分布，有异常值时的中心趋势 |
| 标准差 (Std) | `SQRT(SUM((x-μ)²)/N)` | 离散程度 |
| 四分位数 (Q1/Q3) | 排序后 25%/75% 位置 | 数据分布范围，箱线图基础 |
| IQR | `Q3 - Q1` | 异常值检测阈值 |
| 偏度 (Skewness) | 分布不对称性 | 判断数据是否正态 |
| 峰度 (Kurtosis) | 分布尾部厚度 | 识别厚尾/薄尾分布 |

**异常值判定规则**:

| 方法 | 公式/逻辑 | 适用场景 | 优势 |
|------|----------|---------|------|
| **IQR 法** | `value < Q1 - 1.5×IQR` 或 `value > Q3 + 1.5×IQR` | 偏态分布 | 无分布假设，鲁棒 |
| **Z-score 法** | `|z| > 3`（z = (x-μ)/σ） | 正态分布 | 简单直观 |
| **MAD 法** | `|x - median| / MAD > 3`（MAD = median(|x - median|)） | 含异常值样本 | 比 Z-score 更鲁棒，崩溃点 50% |
| **DBSCAN** | 基于密度的空间聚类，低密度点 = 异常 | 多维异常检测 | 不需要预设异常数量，可发现任意形状 |
| **时间序列异常** | 移动平均残差法：`|actual - MA(n)| > k × σ` | 时间序列数据 | 考虑时间上下文 |

**异常值处理决策**:
```
检测到异常值 → 分类判断:
├── 数据错误（负数价格、未来日期）→ 修正或剔除
├── 真实极端值（大额订单、黑天鹅事件）→ 保留并标注
└── 不确定 → 保留，记录在分析 limitations 中

### 2. 趋势分析

```
分析维度:
├── 时间趋势: 按日/周/月聚合，观察方向
├── 环比: (本期 - 上期) / 上期 × 100%
├── 同比: (本期 - 去年同期) / 去年同期 × 100%
├── 移动平均: 平滑短期波动，识别长期趋势
│   - SMA(n): 简单移动平均，窗口 n
│   - EMA(n): 指数移动平均，权重递减
└── 趋势显著性: Mann-Kendall 检验（非参数）
```

**趋势判定标准**:
- 连续 3 期同向变化 → 趋势信号
- 环比变化 > 10% → 显著变化（需结合业务判断）
- 明显拐点 → 标注可能原因

### 3. 对比分析

```
对比维度:
├── 时间对比: 环比/同比
├── 分组对比: 按维度分组（地区/产品/渠道）
├── 基准对比: 与目标值/行业均值对比
└── 排名分析: Top N / Bottom N

效应量指标:
├── Cohen's d: 两组均值差异 (0.2小/0.5中/0.8大)
├── 百分比差异: (A-B)/B × 100%
└── 相对风险: 发生率比值
```

### 4. 相关性分析

| 方法 | 适用数据类型 | 阈值 |
|------|-------------|------|
| Pearson r | 连续-连续（线性） | \|r\|>0.7 强, >0.4 中, <0.2 弱 |
| Spearman ρ | 有序/非正态 | 同上 |
| Cramer's V | 分类-分类 | >0.3 强, >0.1 中 |
| 互信息 (MI) | 非线性关系 | 相对值比较 |

**注意**: 相关性 ≠ 因果性。显著相关 (p<0.05) 只说明统计上非零。

### 5. 分布分析

```
分析方法:
├── 直方图: 观察分布形态
├── Q-Q 图: 正态性视觉检验
├── Shapiro-Wilk: 正态性统计检验 (p<0.05 → 非正态)
├── 核密度估计 (KDE): 平滑分布曲线
└── 多峰检测: 子群体识别
```

## 洞察提炼规则

分析完成后必须提供文字解读:

1. **数据特征**: 中心在哪？离散多大？有无异常？
2. **变化方向**: 上升还是下降？速度快慢？
3. **显著差异**: 哪些组间差异有统计意义？
4. **业务含义**: 数据特征对业务意味着什么？
5. **建议行动**: 基于分析结果的下一步建议

## 依赖

- skills/data-query — 获取分析数据
- skills/data-clean — 分析前清洗（异常值、缺失值）
- skills/visual — 分析结果可视化
- skills/model — 高级建模（归因、预测、聚类）

## 注意事项

1. 样本量 < 30 时慎用参数检验，使用非参数替代
2. 多重比较时使用 Bonferroni 校正或 FDR
3. 异常值不自动删除 — 先分析原因再决定处理方式
4. 相关分析前先画散点图 — Anscombe's quartet 提醒
