# 路由决策测试用例规格说明

> 版本: 2.0 (2026-04-30)
> 测试目标: 验证 session-routing.md 路由规则的正确性、稳定性、边界覆盖
> 测试方式: AI 行为信号推断（路由为模型内部推理，通过响应行为反向验证）

---

## 行为信号映射

| 观测信号 | 推断路由 | 检测模式 |
|----------|----------|----------|
| Spawn 子 Agent (danalyzer) | DATA_DEEP | spawn / delegat / 交给.*分析 / danalyzer / 子 Agent |
| Spawn 子 Agent (research) | RESEARCH | spawn / delegat / research / 研究报告 |
| 直接回答 | DATA_SIMPLE / GENERAL / COMMAND | 无委托行为，直接给出结果 |
| AskUserQuestion | 模糊 / 低置信度 / 澄清 | 请问 / 哪个口径 / 什么范围 / 具体指 |
| 拒绝/挂起 | 红牌命中 | 无法 / 拒绝 / 需要授权 / 建议缩小 |
| 沿用上轮 | 上下文延续 | 无重新路由信号，继续上轮分析 |
| Reroute 输出 | 重路由 | [reroute: xxx] |
