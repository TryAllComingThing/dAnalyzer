# 路由测试方案 — session-routing.md

> 测试目标：验证路由规则的正确性、稳定性、边界覆盖
> 测试方式：AI 行为信号推断（路由决策为模型内部推理，通过输出行为反向验证）
> 行为信号映射：

| 观测信号 | 推断路由 | 匹配模式 |
|----------|----------|----------|
| 委托子 Agent 执行 | DATA_DEEP / RESEARCH | spawn / delegat / 让.*分析 / 交给 / 子 Agent |
| 直接回答 | DATA_SIMPLE / GENERAL / COMMAND | 无委托行为，直接给出结果 |
| 反问澄清 | 模糊 / 低置信度 / 红牌未达阻断 | AskUserQuestion / 请问 / 哪个口径 / 什么范围 |
| 拒绝/挂起 | 红牌命中拦截 | 无法 / 拒绝 / 需要授权 / 建议缩小 |
| 沿用上轮方式 | 上下文延续 | 无重新路由信号，继续上轮分析 |

## 分类 1：正常路径 — 每个意图类型

### R-DEEP-01: DATA_DEEP 典型请求

```
输入: 帮我分析销售数据，先清洗再做趋势分析，最后生成可视化看板
预期路由: DATA_DEEP → Spawn danalyzer
验证信号: 委托子 Agent、启动分析流程、提及清洗/趋势/看板步骤
```

### R-DEEP-02: DATA_DEEP 建模请求

```
输入: 对用户数据做 RFM 分群，输出各群特征
预期路由: DATA_DEEP → Spawn danalyzer
验证信号: 委托执行、提及 RFM 建模
```

### R-RESEARCH-01: RESEARCH 典型请求

```
输入: 写一份关于物流行业数字化转型的研究报告
预期路由: RESEARCH → Spawn research
验证信号: 委托 research agent（或提示该功能预留中）
```

### R-SIMPLE-01: DATA_SIMPLE 单次查询

```
输入: 上个月的总销售额是多少
预期路由: DATA_SIMPLE → 主会话直接执行
验证信号: 直接查询并返回结果，不委托子 Agent
```

### R-SIMPLE-02: DATA_SIMPLE 简单聚合

```
输入: 统计一下各部门的人数
预期路由: DATA_SIMPLE → 主会话直接执行（自动降级）
验证信号: 直接执行简单聚合，不 spawn agent
```

### R-GENERAL-01: GENERAL 编程请求

```
输入: 帮我写一个 Python 脚本来计算文件 MD5
预期路由: GENERAL → 主会话
验证信号: 直接编写代码，不触发数据分析流程
```

### R-GENERAL-02: GENERAL 日常对话

```
输入: 什么是假设检验中的 p 值
预期路由: GENERAL → 主会话
验证信号: 直接解释概念，不涉及数据操作
```

### R-COMMAND-01: COMMAND 系统指令

```
输入: /help
预期路由: COMMAND → 主会话
验证信号: 显示帮助信息
```

## 分类 2：边界与异常

### R-AMBIGUOUS-01: DEEP vs RESEARCH 模糊

```
输入: 分析用户行为模式并输出完整报告
预期路由: 模糊 → AskUserQuestion 澄清
验证信号: 反问用户具体需求方向（侧重分析还是报告）
```

### R-AMBIGUOUS-02: 置信度低（请求过于模糊）

```
输入: 帮我看看数据
预期路由: 低置信度 → AskUserQuestion 澄清
验证信号: 反问具体想看什么数据、什么维度
```

### R-REDFLAG-01: 写表/删库操作

```
输入: 把清洗后的数据写回生产数据库
预期路由: 红牌命中 → 挂起 AskUserQuestion
验证信号: 拒绝或要求二次授权，不直接执行
```

### R-REDFLAG-02: 预测无历史数据

```
输入: 预测明年的销售额
预期路由: 红牌命中 → 驳回/要求补充数据
验证信号: 反问历史数据范围或回溯窗口
```

### R-REDFLAG-03: 口径模糊

```
输入: 分析用户活跃度
预期路由: 红牌命中 → 确认口径
验证信号: 反问 DAU/MAU/登录次数等具体口径
```

### R-REDFLAG-04: 大范围扫描

```
输入: 分析过去三年的全量访问日志
预期路由: 红牌命中 → 建议缩小范围或抽样
验证信号: 提示数据量大、建议缩小时间范围或抽样
```

### R-REDFLAG-05: 伪分析

```
输入: 我这里有 3 个数字，帮我做深度分析
预期路由: 红牌命中 → 澄清/仅做简单解读
验证信号: 提示数据量不足，仅做有限解读
```

## 分类 3：自动降级

### R-DEGRADE-01: 复杂表述但实际简单

```
输入: 我要做一个全面的数据分析，先看看这个月每天的订单量趋势
预期路由: DATA_DEEP 表述 → 复杂度校验 → 降级为 DATA_SIMPLE
验证信号: 主会话直接处理单次查询，不 spawn agent
```

### R-DEGRADE-02: 明确不需要图表

```
输入: 帮我算一下平均客单价，不用出图
预期路由: 降级为 DATA_SIMPLE
验证信号: 直接计算返回数值，不生成图表
```

## 分类 4：上下文延续

### R-CONTEXT-01: 继续上轮分析

```
轮次1 输入: 帮我分析 1 月销售数据
轮次1 预期: DATA_DEEP → Spawn danalyzer

轮次2 输入: 继续
轮次2 预期: 沿用上轮路由，不重新路由
验证信号: 继续分析而非重新询问需求
```

### R-CONTEXT-02: 深化分析

```
轮次1 输入: 各品类销售额排名
轮次1 预期: DATA_SIMPLE → 主会话查询

轮次2 输入: 再分析一下增长最快的品类
轮次2 预期: 沿用上轮路由
验证信号: 基于上轮结果深入，不重新路由
```

### R-CONTEXT-03: 话题切换重置

```
轮次1 输入: 分析销售数据
轮次1 预期: DATA_DEEP

轮次2 输入: 帮我写个 SQL 查询用户表
轮次2 预期: 新话题 → 重置 → DATA_SIMPLE
验证信号: 不继续分析销售，转而处理 SQL 查询
```

## 分类 5：多意图冲突仲裁

### R-CONFLICT-01: 深度分析优先

```
输入: 查一下订单数，再做个 RFM 分群
预期路由: 深度分析优先 → DATA_DEEP
验证信号: 委托子 Agent 执行 RFM（含简单查询作为前置步骤）
```

### R-CONFLICT-02: 简单 + 简单 = 按表从上到下

```
输入: 查一下用户数和订单数
预期路由: DATA_SIMPLE（表顺序优先）
验证信号: 主会话直接执行两个简单查询
```

## 分类 6：主会话执行边界

### R-BOUNDARY-01: 主会话不越权

```
DATA_SIMPLE 路由后 → 主会话只做简单查询
验证信号: 不启动多步清洗、建模或预测
```

### R-BOUNDARY-02: Spawn 后主会话不抢活

```
DATA_DEEP 路由 → 委托子 Agent
验证信号: 主会话等待子 Agent 返回，不做预分析
```

---

## 分类 7：Spawn 验证

### R-SPAWN-01: 子 Agent 返回验证

```
DATA_DEEP 路由 → 子 Agent 执行完成
验证信号: 主会话检查输出文件存在、非空、不含 PII
```

```
输入: 对比电商和物流部门的数据差异
预期路由: DATA_DEEP → Spawn danalyzer（跨源对比分析）
```

### R-COMPLEX-04: 行业术语分析

```
输入: 计算 LTV 和 CAC，再做个同期群分析
```

## 分类 9：语言与风格变体

### R-STYLE-01: 纯英文查询

```
输入: show me last month total sales
预期路由: DATA_SIMPLE → 主会话（单次聚合查询，语言不影响意图）
```

### R-STYLE-02: 中英混合

```
输入: 分析一下昨天的 order data，做个 trend chart
预期路由: DATA_DEEP（"分析"+"trend chart"= 深度分析）
```

### R-STYLE-03: 极短输入

```
输入: 数据
预期路由: 模糊 → AskUserQuestion 澄清（单次词意图不明）
```

### R-STYLE-04: 空输入/标点

```
输入: ???
预期路由: GENERAL → 主会话（非分析类输入）
```

### R-STYLE-05: 明确指定 Agent

```
输入: 用 danalyzer 分析用户数据
预期路由: DATA_DEEP → Spawn danalyzer（用户显式指定路由目标）
```

### R-STYLE-06: 明确要求不 Spawn

```
输入: 就在当前会话分析，不要开子 Agent
预期路由: 强制 DATA_SIMPLE → 主会话（用户拒绝 spawn，即使分析较复杂也应尊重）
```

### R-STYLE-07: 长段落请求

```
输入: 我们部门最近在做年度总结，需要分析一下过去一年的销售数据，
     包括月度趋势、品类排名、区域对比，还要预测 Q4 的销售额。
     另外如果有异常波动的月份需要标注出来。最后生成一份 PPT 报告。
预期路由: DATA_DEEP → Spawn danalyzer（多步骤深度分析）
```

```
输入: 销售额多少？用户数多少？客单价多少？
```

### R-LIMIT-05: 矛盾请求

```
输入: 简单看一下数据，但要出完整分析报告
```

### R-LIMIT-06: 闲聊中嵌入分析

```
输入: 今天天气不错，顺便帮我查一下昨天的订单量
```

### R-LIMIT-07: 元问题（关于路由本身）

```
输入: 刚才的路由决策是什么
```

### R-LIMIT-08: 行业黑话请求

```
输入: 做一下流失预警和用户分层
```
